Frequently Asked Questions

What is Ronin

What language is Ronin written in?

Ruby.

What version of Ruby should I run Ronin on?

We recommend running Ronin on Ruby 1.9.3-p0

Ronin can run on:

  • Ruby: >= 1.9.2, 1.8.7
  • JRuby: >= 1.5.1
  • Rubinius: >= 1.2.4

However, Ronin does not support Ruby 1.8.6, due it's poor performance and incompatibilities. If you are still using 1.8.6, we strongly recommend that you upgrade to either Ruby 1.8.7 or 1.9.2.

What is the easiest way to install Ruby?

Ruby Version Manager (RVM)

What are some good resources for learning Ruby?

LearnToProgram is an excellent book and tutorial which provides a solid introduction to Ruby.

There's also the free and more technical Pragmatic Programmer's Guide to Ruby, which many Rubyists first learned from.

If you're more of a hands-on learner, there is also the Try Ruby in your Browser (Beta) interactive tutorial. If your still wanting more hands-on challenges, try Ruby Koans.

Is Ronin a Framework?

Not exactly. Ronin can be thought of more as an Environment or Platform for code, exploits, payloads or research. Ronin provides the Convenience Methods, Database, libraries and a Repository System to package it all up.

One of Ronin's design goals was to create a Platform that facilitated and encouraged third-party work.

What is Ronin licensed under?

GPL-3

Additionally, the ronin-support library is licensed under the LGPL-3 license.

How well documented is Ronin?

Ronin has extensive YARD documentation covering it's API. Additional guides and screencasts are always in the works.

Does Ronin have a command-line interface (CLI)?

Why yes. Ronin's design allows for the libraries of Ronin to add sub-commands to the main ronin command. To view all available commands simply run the following command: 

$ ronin help

To view the help information of a specific command:

$ ronin help command_here

Exploitation

Does Ronin have builtin exploits?

Ronin does not ship with any builtin Exploits. Although, Ronin makes it rather easy to write your own exploits using ronin-exploits library and distribute them in Repositories.

It takes a lot of time and energy to write and weaponize exploits for known vulnerabilities; let alone find new vulnerabilities. So, we decided to leave the exploit writing up to the skilled security researchers of the world, and let them decide how and when to distribute their own exploits.

Does Ronin provide an autopwn-like feature?

Ronin does not provide an autopwn-like feature. Features such as autopwn make assumptions about the workflow of a "real attacker", and attempt to automate it. Naturally, the "real attackers" will change their tactics and use new types of information gathering (ex: Datamining Social Networks) and attacks (ex: Phishing) which may not cleanly fit into existing Exploitation Frameworks; thus rendering autopwn-like features outdated and ineffective.

Ronin was designed to not impose a workflow onto the user. Instead, Ronin provides useful APIs, commands and a Database for the user to leverage however they want to.

Repositories

What is a Ronin Repository?

Ronin Repositories are very similar to RubyGems. They both can contain Ruby code in a lib/ directory, command-line scripts in a bin/ and data files in a data/ directory. Ronin Repositories also have a ronin.yml metadata file and a cache/ directory for exploits, payloads, scanners or anything else Ronin can cache into it's Database.

Think of Ronin Repositoires as a quicker / less-formal way to organize your code.

How are Ronin Repositories hosted?

A Ronin Repository can be hosted on any Subversion (SVN), Mercurial (Hg), Git or Rsync server.

One could easily host a Ronin Repository on free source-code hosting sites such as, GitHub, Gitorious, Google Code or SourceForge.

How can I install a Ronin Repository?

Simply install the URL of the Ronin Repository using the ronin-repos command: 

$ ronin-repos --install svn://www.example.com/repo

If you have previously checked out a Ronin Repository, then simply add the path of the local Repository using the ronin-repos command: 

$ ronin-repos --add path/to/local/repo

How can I generate a skeleton Ronin Repository?

You can generate a skeleton Ronin Repository by simply running the following command provided by the ronin-gen library: 

$ ronin-gen repository path/to/repo

If you wish to specify additional Repository metadata:

$ ronin-gen repository -n Repo-Name -L GPL-3 \
  -m "Author " \
  -S http://svn.mysite.com/repo \
  -U http://myblog.com \
  -D "This is my Repo, enjoy." \
  path/to/repo

Development

How readable is Ronin's code-base?

Other projects have been criticized for having poor programming conventions or a lack of documentation. To avoid this, Ronin follows the Ruby community's style guidelines to ensure that all source-code is clean, readable and maintainable.

Where is Ronin's code-base hosted?

Ronin and it's many libraries are hosted on GitHub.

Does Ronin have a test-suite so that I can ensure that Ronin is bug free?

Yes, Ronin comes with extensive RSpec specification tests.

One can run the specification suite in any of Ronin's libraries, by running the following command: 

$ rake spec
